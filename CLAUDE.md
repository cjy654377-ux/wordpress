## 1. 토큰 최적화 — 핵심 원칙

### 1.1 응답 규칙
- **간결 우선** — 단순 질문 1~2문장, 코드 변경분만 출력
- **선언 금지** — "먼저 ~하겠습니다" 등 계획 선언 없이 바로 실행
- **앵무새 금지** — 사용자 요청 반복하지 않음
- **마무리 인사 금지** — "추가 질문 있으시면~" 등 관용구 제거
- **도구 결과 재설명 금지** — Grep/Read 결과 나왔으면 바로 다음 단계

### 1.2 서브에이전트 모델 정책
| 모델 | 용도 | 비용 비율 |
|------|------|----------|
| **haiku** | 단순 탐색, 파일 검색, 간단한 검증 (기본값) | 1x |
| **sonnet** | 코드 작성, 조사, 분석 등 일반 작업 | 5x |
| **opus** | 계획 수립, 아키텍처 설계, 치명적 오류 해결만 | 25x |

**원칙**: haiku로 될 일에 sonnet 쓰지 말 것. opus는 최후의 수단.

### 1.3 도구 호출 최적화
- **병렬 호출 필수** — 독립적인 도구 호출은 반드시 한 번에 묶어서
- **Edit > Write** — 기존 파일 수정은 Edit, 새 파일만 Write
- **Glob/Grep > Bash** — 파일 검색에 find/grep 대신 전용 도구
- **Read offset/limit** — 500줄+ 파일은 부분 읽기
- **재읽기 금지** — 이미 읽은 파일을 같은 세션에서 다시 읽지 않음

### 1.4 콘텐츠 생성 최적화
- **로직/콘텐츠 분리** — `engine.py`(로직) + `template.html`(구조) + `data.json`(데이터)
- **반복 블록 금지** — 동일 CSS/HTML 2회 이상 작성 → 템플릿화
- **공통코드 1회 작성** — 인증/API 호출 등은 engine.py에서 import
- **이미지**: Python PIL 사용, 1200x630, HelveticaNeue 폰트

## 2. AdSense 수익화 전략

### 2.1 핵심 원칙
이 프로젝트의 목적은 **AdSense를 통한 수익 창출**이다. 모든 콘텐츠 결정은 수익 극대화 관점에서 판단한다.
- 주제 선정 시 **검색량(트래픽) + CPC(클릭 단가)** 둘 다 고려
- 트렌드 뉴스(단기 트래픽 폭발) + 에버그린 콘텐츠(장기 안정 트래픽) 균형
- 매 세션마다 **수익에 가장 효과적인 다음 행동**을 제안할 것

### 2.2 고수익 주제 우선순위 (CPC 기준)
| 순위 | 카테고리 | 예상 CPC | 전략 |
|------|----------|----------|------|
| 1 | Finance/Insurance/Legal | $2-10+ | 한국 관련 금융·보험 정보를 영문으로 |
| 2 | Technology/SaaS | $1-5 | K-Tech, 한국 앱/서비스 리뷰 |
| 3 | Health/Beauty | $1-3 | K-Beauty 제품 리뷰, 루틴 가이드 |
| 4 | Travel | $0.5-2 | 한국 여행 가이드, 맛집 |
| 5 | Entertainment | $0.3-1 | K-Drama, K-Pop (트래픽↑ CPC↓) |
| 6 | Breaking News | $0.1-0.5 | 바이럴 트래픽용 (CPC 낮지만 볼륨 큼) |

### 2.3 콘텐츠 품질 (AdSense 승인 + 유지 필수)
- **최소 800단어** — thin content 절대 금지
- **H2 3개 이상** — 테이블/리스트 포함
- **독창적 분석** — 일반 정보 나열 금지, 구체적 수치/데이터 필수
- **실용성** — 가격, 위치, 시간 등 행동 가능한 정보
- **SEO** — 제목에 키워드, 자연스러운 분산, excerpt 활용
- **내부 링크** — 관련 글 간 상호 링크로 세션 시간 증가

### 2.4 모바일 가독성 (필수 유지)
트래픽의 60-80%가 모바일이므로 모든 콘텐츠는 모바일 최적화 필수:
- **한 문단 3줄(약 120자) 이내** — 모바일에서 화면 가득 차는 블록 금지
- **max-width: 100%** — 테이블, 이미지 반응형 필수
- **폰트 16px 이상** — 작은 글씨 금지
- **터치 타겟 44px 이상** — 링크/버튼 간격 확보
- **CSS 규칙**:
  - `.info-table { display: block; overflow-x: auto; }` (모바일 테이블 스크롤)
  - `img { max-width: 100%; height: auto; }` (이미지 반응형)
  - 인라인 스타일에 `width: 100%` 대신 `max-width` 사용
- **광고 배치 고려** — 콘텐츠 사이 여백 확보 (AdSense 자동 광고 삽입 영역)

### 2.5 카테고리 전략
| 카테고리 | 목적 | 발행 비중 |
|----------|------|----------|
| K-Beauty | 고CPC + 에버그린 | 25% |

### 2.6 발행 워크플로우
```

2. 상세 리서치 (WebSearch)

```
